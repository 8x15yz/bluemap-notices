## 2. 다음 개발 방향

### Phase 1. 로컬 실데이터 연결

#### Codex가 자동으로 할 수 있는 작업

- [x] `.env.example` 기준으로 필요한 환경변수 목록을 점검한다.
- [x] `.env`가 준비되면 `npm.cmd run db:init`을 실행해 DB 스키마를 적용한다.
- [x] `.env`가 준비되면 `npm.cmd run dev`로 개발 서버를 실행한다.
- [x] 나라장터 실데이터 수집 API를 호출해 수집 결과를 확인한다.
- [x] 후보 공고가 Postgres에 저장되는지 SQL로 확인한다.
- [x] 점수 0점 공고가 웹 목록에 노출되지 않는지 확인한다.
- [x] 마감일이 지난 공고가 후보에서 제외되는지 확인한다.
- [x] 확인된 항목은 이 문서의 체크박스를 `[x]`로 갱신한다.

#### 사용자가 해야 할 작업

- [x] Postgres 접속 정보를 준비한다.
  - 로컬에서 테스트하려면 Postgres를 설치하거나 Docker로 Postgres 컨테이너를 실행한다.
  - 클라우드를 쓰려면 Supabase, Neon, Railway, Render 등에서 Postgres 프로젝트를 만든다.
  - 접속 문자열은 `postgres://USER:PASSWORD@HOST:PORT/DB_NAME` 형식으로 준비한다.
- [x] `.env` 파일을 생성한다.
  - 프로젝트 루트에서 `.env.example`을 복사해 `.env`를 만든다.
  - `.env` 파일은 git에 올리지 않는다.
- [x] `DATABASE_URL`을 `.env`에 입력한다.
  - 준비한 Postgres 접속 문자열을 `DATABASE_URL=` 뒤에 붙여넣는다.
  - 클라우드 DB가 SSL을 요구하면 `PGSSLMODE=require`도 추가한다.
- [x] 나라장터 OpenAPI 서비스 키를 발급받는다.
  - 공공데이터포털에 로그인한다.
  - 나라장터 공공데이터개방표준서비스 활용 신청을 완료한다.
  - 나라장터 입찰공고정보서비스도 함께 활용 신청한다.
  - 두 서비스가 각각 승인 상태인지 확인한다.
  - 승인된 서비스 키를 복사한다.
- [x] `G2B_SERVICE_KEY`를 `.env`에 입력한다.
  - 인코딩 키 또는 일반 인증키 중 하나를 넣는다.
  - 현재 코드는 일반 인증키도 자동 URL 인코딩한다.
  - 공공데이터개방표준서비스와 입찰공고정보서비스 모두 같은 `G2B_SERVICE_KEY`를 사용한다.
  - 두 API의 분기는 키가 아니라 `G2B_STANDARD_API_BASE_URL`, `G2B_BID_PUBLIC_API_BASE_URL`로 한다.
